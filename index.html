kdf.sh is a short domain that is on the home row of the standard qwerty layout.

kdf.sh is a mix of utilities, experiments, and things that don't have a dedicated domain yet.

kdf.sh is self-referential.

kdf.sh hosts:

  • gritonputty, a typing tutor aimed at the twiddler.
  • Various mini-http utilities:
    • Http status codes, including:
      • http://200.kdf.sh/content
      • http://404.kdf.sh/oh noes
      • Other assorted status codes.
      • http://301.kdf.sh/http://200.kdf.sh/magic redirect
    • http://time.kdf.sh; see also /EST; /unix; /unix/1234567890/EST; and /1971-01-01T12:00:00Z/unix
    • http://echo.kdf.sh, which echos back the full request it received.
    • http://ip.kdf.sh, a minimalist "what is my ip" service.
    • http://proxy.kdf.sh, a HTTP proxy that can add headers to the request or response with "?@" and "?!"; e.g., http://proxy.kdf.sh/http://echo.kdf.sh/body?@Request-Header=magic?!ReponseHeader=more magic
  • Various mini-DNS utilities:
    • prefix.data.127.0.0.1.ip.kdf.sh resolves to 127.0.0.1. (Inspired by xip.io)
    • www.google.com.cname.kdf.sh resolves to a CNAME for google.com (though Google's servers don't respond to that hostname).
    • me.ip.kdf.sh resolves to a host requesting the DNS entry. Due to caching, this is not actually very useful.
  • Various other secret, obscure, or not-yet documented things.
